Neuropath — Live Scraping + LLM Pipeline Fix
Problem Summary
When a user clicks "Yes, I Know my passion & skill" and enters a field, the app navigates to courses/scholarships pages — but those pages show static/stale fallback data instead of live internet-sourced results. The root causes are:

Scrapers silently fail: NPTEL, Swayam, MIT OCW, Unstop, Devpost all use client-side JavaScript rendering — axios + cheerio only gets empty HTML shells. Every scrape returns 0 results and silently falls back to hardcoded or LLM data.
LLM fallback uses stale training data: The current NVIDIA Qwen prompts ask the model to recall data from its training, not to search the internet. No web URLs are retrieved.
No MongoDB persistence of results per user: Scraped/LLM results are only stored in a 30-min in-memory cache. After a server restart, everything is lost and re-fetched from scratch.
Scraping is triggered on component mount, not when the user submits their passion. So there's no head-start and the user sees a loading spinner every time.
Proposed Solution Architecture
User enters passion
       │
       ▼
Backend: POST /api/career/enrich?field=<field>  ← triggered immediately on passion submit
       │
       ▼
[Step 1] LLM (Qwen) asked: "What are the best URLs on Naukri, LinkedIn, Unstop, 
          Coursera, NSP, AICTE, etc. for <field> right now?"
       │
       ▼
[Step 2] Axios fetches those LLM-recommended URLs + fixed reliable sources
         (Google Scholar links, OpenAlex API, Coursera search API, NSP gov API)
       │
       ▼
[Step 3] Cheerio parses scraped HTML for courses, scholarships, institutions, jobs
       │
       ▼
[Step 4] LLM synthesises & enriches: takes raw scraped text → structured JSON
       │
       ▼
[Step 5] Result stored in MongoDB collection: career_data
         { field, courses[], scholarships[], institutions[], training[], 
           liveLinks[], userId, createdAt }
       │
       ▼
Frontend fetches from GET /api/career/data?field=<field>
(returns DB-cached result if fresh < 6h, else re-triggers enrich)

IMPORTANT

The scraper uses the LLM to recommend URLs (not scrape Twitter/LinkedIn directly — those require auth). Instead, it scrapes:

Naukri.com job listings search pages (public)
Unstop.com competitions (public)
Coursera public search pages
NSP (scholarships.gov.in) — public government portal
AICTE — public
LinkedIn public job pages (non-authenticated search results)
Google News RSS for trending field news
This is all done via legitimate public HTTP requests, same as a web browser would.

WARNING

Many sites use JavaScript rendering. For sites that block cheerio-only scraping, we'll use:

Their search/API endpoints (JSON responses) instead of HTML
Google News RSS feed as a reliable trending source
LLM synthesis of scraped text even if partial
